# 总体设计说明书

## 技术栈
- 前端：Vue 3 + TypeScript + Vite（Canvas/SVG 可视化）。
- 后端：Spring Boot 3 + Maven（Web + Validation）。
- 数据库：MySQL（后续接入 MyBatis-Plus）。

## 架构分层
- 前端
  - 页面：主页/算法目录/动画播放器/对比页/数据编辑器/登录注册/个人中心。
  - 模块：`algorithm-engine`（步骤生成）、`visualizer`（渲染）、`player`（时间轴控制）、`editor`（输入）、`analytics`（统计）。
- 后端
  - 控制器：认证、用户、算法、运行记录、收藏、统计、健康检查。
  - 服务与数据层：ORM 与 MySQL，提供持久化与查询接口。

## 统一协议（Step/State）
- Step：演示过程的原子操作（如 `compare | swap | visit | enqueue | dequeue | relax`），携带索引/节点/边与元数据。
- StateSnapshot：当前可视化所需的结构快照（数组、图、树/堆等），用于渲染与回退。

## 关键模块职责
- Algorithm（策略模式）：对输入生成步骤序列；提供 `init/next/hasNext/getState`。
- Visualizer（渲染器）：根据 Step/State 执行差量更新与高亮。
- Player（时间轴控制）：播放、暂停、单步、回退、速度调整。
- EventBus（可选）：解耦 UI 与引擎，实现订阅通知。

## 部署与跨平台
- 响应式设计与触控支持，PWA 可选。
- 前后端分离部署，Nginx 反向代理与 HTTPS。
