# 需求规格说明书

## 项目目标
- 深度理解并以交互式动画演示经典算法的运行原理与状态变化。
- 满足跨平台访问（Web/移动端友好），可交互、可扩展、可部署的课程项目。
- 后端与数据库用于用户管理、算法元数据与运行记录的持久化。

## 用户与角色
- 普通用户：浏览算法目录、运行演示、保存运行记录与收藏。
- 管理员：维护算法目录与说明，查看统计，必要时管理用户与内容。

## 主要功能
- 算法动画演示：播放/暂停、单步、回退、速度控制；过程高亮（比较/交换/访问等）。
- 输入编辑器：数组/图/树的输入与参数配置，支持随机生成与自定义。
- 记录与复现：保存运行输入与参数（可选保存步骤），支持回放与统计。
- 对比演示：并排比较同类算法的步骤数、比较次数与耗时。

## 非功能需求
- 代码规范与可维护性：前端 ESLint+Prettier；后端 Checkstyle/Spotless。
- 性能：差量渲染、批量绘制、避免全量重绘；响应式布局。
- 安全：JWT 登录、权限控制、输入校验、CORS 配置。
- 部署：基于 Web 的部署与运行，提供可预览链接。

## 验收标准（对齐课程设计）
- 第一阶段：完成需求分析与分工，原型交互流程确定。
- 第二阶段：两次验收——快速原型（排序演示与控制）、整体功能（图算法、持久化、对比模式）。
- 第三阶段：答辩展示与总结报告。

## 评分映射
- 理论与总结报告（40%）：资料查阅、问题分解与验证标准、实现细节与测试方案、过程总结。
- 功能实现（40%）：核心动画、交互与持久化、技术采用合理性与先进性、代码规范。
- 答辩（20%）：完整展示与正确回答问题。
